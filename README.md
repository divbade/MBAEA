# MBAEA — MBA Executive Assistant

Your intelligent executive assistant for MBA life. Extract commitments from unstructured text, review and prioritize them, generate weekly plans with deterministic reasoning, and sync to Google Calendar.

## Features

- **🎯 Structured Extraction** — Paste any text and our AI extracts commitments with type, action, goal tags, confidence, and rationale
- **📥 Smart Inbox** — Ranked candidates with inspectable scoring (goal weights × urgency × confidence)
- **📅 Deterministic Planner** — Weekly plan generated by heuristic engine (not LLM), with reasoning for each block
- **👍 Feedback Learning** — Thumbs up/down adjusts your goal weights automatically
- **🔗 Google Calendar** — Read events and write plan blocks (with MBAEA tags)
- **🎨 Demo Mode** — Full demo with seed data, no Google account needed

## Tech Stack

- **Frontend**: Next.js 16 (App Router) + TypeScript + Tailwind CSS v4
- **Backend**: Next.js API Routes
- **Database**: Supabase (Postgres + Auth + RLS)
- **AI**: OpenAI GPT-4o-mini (structured JSON extraction)
- **Calendar**: Google Calendar API
- **Hosting**: Vercel

## Quick Start

### Prerequisites

- Node.js 18+
- A [Supabase](https://supabase.com) project
- An [OpenAI API key](https://platform.openai.com/api-keys)
- A [Google Cloud](https://console.cloud.google.com) project with Calendar API enabled (optional for demo)

### 1. Install Dependencies

```bash
cd MBAEA
npm install
```

### 2. Set Up Environment Variables

Copy the example file and fill in your keys:

```bash
cp .env.local.example .env.local
```

Edit `.env.local` with your actual values:

```env
NEXT_PUBLIC_SUPABASE_URL=https://xxxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJhbGci...
SUPABASE_SERVICE_ROLE_KEY=eyJhbGci...
OPENAI_API_KEY=sk-proj-...
GOOGLE_CLIENT_ID=xxxxx.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=GOCSPX-...
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

### 3. Set Up Supabase Database

1. Go to your Supabase dashboard → **SQL Editor**
2. Copy and run the contents of `supabase/migrations/001_schema.sql`
3. Enable **Google Auth** under Authentication → Providers → Google

### 4. Run Locally

```bash
npm run dev
```

Visit [http://localhost:3000](http://localhost:3000)

### 5. Seed Demo Data (Optional)

After signing in for the first time, run the seed script in Supabase SQL Editor:

```sql
-- Copy contents of supabase/seed.sql and run it
```

This creates 5 demo candidate items for testing.

## Demo Flow

1. **Sign in** with Google at `/login`
2. **Onboarding** — select goals (recruiting, academics, health...) and set constraints
3. **Capture** — paste text like "Coffee chat with McKinsey recruiter Friday 3pm"
4. **Inbox** — review extracted candidates, accept/edit/dismiss with rank scores
5. **Planner** — generate weekly plan, click blocks to see reasoning, give feedback
6. **Settings** — adjust goal weights (automatically updated by feedback)

## Deploy to Vercel

1. Push to GitHub
2. Import in [Vercel](https://vercel.com/new)
3. Add all environment variables from `.env.local`
4. Update `NEXT_PUBLIC_APP_URL` to your Vercel domain
5. Add `https://your-app.vercel.app/api/google/callback` to Google OAuth redirect URIs
6. Deploy!

## Security

- All tables have RLS policies (`user_id = auth.uid()`)
- Google API tokens stored server-side only (no client RLS policies)
- Service role key never exposed to client
- OAuth tokens exchanged server-side
